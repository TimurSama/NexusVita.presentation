# База данных на Vercel

## Важно: SQLite не работает на Vercel

SQLite - это файловая база данных, которая требует постоянного доступа к файловой системе. Vercel использует serverless функции, которые не поддерживают постоянное хранение файлов.

## Решения для Vercel

### Вариант 1: Vercel Postgres (Рекомендуется)

Vercel предлагает встроенную интеграцию с Postgres:

1. Перейдите в настройки проекта на Vercel
2. Выберите "Storage" → "Create Database" → "Postgres"
3. Получите строку подключения
4. Обновите код для использования Postgres вместо SQLite

### Вариант 2: Внешняя база данных

Используйте внешний сервис базы данных:
- **Supabase** (Postgres) - бесплатный тариф, простой в использовании
- **PlanetScale** (MySQL) - бесплатный тариф
- **Neon** (Postgres) - бесплатный тариф
- **Railway** (Postgres) - бесплатный тариф

### Вариант 3: Vercel KV (Redis)

Для простых данных можно использовать Vercel KV (Redis), но это не полноценная база данных.

## Миграция с SQLite на Postgres

Если вы хотите использовать Postgres, нужно:

1. Установить `pg` или `@vercel/postgres`:
```bash
pnpm add @vercel/postgres
# или
pnpm add pg
```

2. Обновить `server/database.ts` для использования Postgres вместо SQLite

3. Обновить все SQL запросы под синтаксис Postgres

## Текущая ситуация

Сейчас база данных SQLite настроена для локальной разработки. Для продакшена на Vercel нужно использовать один из вариантов выше.

## Рекомендация

Для быстрого старта рекомендую использовать **Supabase**:
- Бесплатный тариф
- Простая настройка
- Автоматические миграции
- Встроенная аутентификация
- Хорошая документация
