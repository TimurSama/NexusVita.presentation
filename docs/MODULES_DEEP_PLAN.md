# Deep Plan for All 7 Health Modules

## Overview
Each module should have: individual profile page, key metrics, functions, information to fill, goals, and plans.

---

## 1. MOVEMENT (Physical Activity)

### Individual Profile Page
- **Physical type**: Endomorph / Mesomorph / Ectomorph
- **Fitness level**: Beginner / Intermediate / Advanced / Athlete
- **Primary goal**: Weight loss / Muscle gain / Endurance / Flexibility / General health
- **Body metrics**: Weight, Height, BMI, Body fat %, Muscle mass
- **Medical restrictions**: Joint problems, Heart conditions, Injuries
- **Preferred activities**: Running, Swimming, Cycling, Gym, Yoga, Sports

### Key Metrics to Track
- Daily steps count
- Active minutes
- Calories burned
- Distance covered
- Heart rate zones
- Workout duration
- Rest days
- Sleep quality (linked to recovery)

### Functions
- Workout planner & scheduler
- Exercise library with videos
- Progress tracking (photos, measurements)
- Integration with fitness devices (Apple Watch, Fitbit, etc.)
- Personal trainer marketplace
- Challenges & competitions
- Recovery recommendations

### Information to Fill Initially
1. Current fitness level assessment
2. Body measurements (chest, waist, hips, arms, legs)
3. Medical history related to physical activity
4. Available equipment (home gym, gym membership, none)
5. Time availability for workouts
6. Preferred workout times
7. Sleep schedule (for recovery planning)

### Goals Examples
- Run 5K in under 30 minutes
- Lose 5 kg in 2 months
- Do 20 push-ups
- Walk 10,000 steps daily
- Complete a marathon

## Technical Implementation Notes

### Database Schema Extensions

```sql
-- Movement profile
CREATE TABLE movement_profiles (
  user_id INTEGER PRIMARY KEY,
  fitness_level VARCHAR(20),
  physical_type VARCHAR(20),
  primary_goal VARCHAR(50),
  body_metrics JSONB,
  medical_restrictions JSONB,
  preferred_activities JSONB
);

-- Nutrition profile
CREATE TABLE nutrition_profiles (
  user_id INTEGER PRIMARY KEY,
  dietary_type VARCHAR(30),
  allergies JSONB,
  eating_habits VARCHAR(30),
  cooking_skills VARCHAR(20),
  budget_range VARCHAR(10),
  cuisine_preferences JSONB
);

-- Similar tables for other 5 modules...
```

### API Endpoints Needed
- `GET /api/users/{id}/profile/{module}` - Get module profile
- `POST /api/users/{id}/profile/{module}` - Create/update profile
- `GET /api/users/{id}/goals/{module}` - Get module goals
- `POST /api/users/{id}/goals/{module}` - Create goals
- `GET /api/users/{id}/metrics/{module}` - Get tracked metrics
- `POST /api/users/{id}/metrics/{module}` - Log metrics

### Frontend Components Needed
- `ModuleProfileForm` - Profile creation/editing
- `ModuleDashboard` - Module-specific dashboard
- `MetricsTracker` - Metric input and visualization
- `GoalsManager` - Goal setting and tracking
- `ProgressCharts` - Visual progress representation
